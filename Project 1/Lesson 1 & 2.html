Important Concepts

Lesson 1: The Basics of HTML/Web

What is the Web?

The web is comprised of three main things. The Browser, Internet, and Servers. In essence, the web is a lot of computers talking to one another. A person sends an HTTP Request to a server which then returns the requested information. HTTP (which stands for Hyper Text Transfer Protocol) is comparable to the highways that connect cities to one another. The Broswer is the individual machine that conects to the network or the Internet and sends requests to the Server


HTML?

HTML stands for Hypte Text Markup Language. It is what gives form and function to web pages. It's made up of two things, Text content and Markup. Text Content makes up what you see in a webpage (content) while Markup decides how that content looks.


Elements and Tags

An Element refers to that which is everything between an opening and closing Tag. There are two types of Elements, Inline and Block. Inline is an Element that just makes a line. An example would be <br>, <br> is a void tag that moves to the next line, or sometimes referred to as a Break. A Block Element creates an invisible Box that has specific height and width restrictions. <p> is a block Element called paragraph. A Tag is always contained within angled brackets. Most tags have an opening tag (<p>) followed by a closing tag (</p>). There are some called Void Tags that do not require closing tags.

Some of the more commonly used tags are as follows:

  1. <b> </b> - The Bold tag.
  2. <em> </em> - The Italics tag.
  3. <a> </a> - This is the anchor tag.
The anchor tag is used to place Hyper Links or Links in the content of a webpage. <a href="www.reddit.com">derp</a> is an example of what can be used to insert a link in a page. It shows up as derp. (href) is the atrribute that is being assigned to the anchor.

Computers are Simple

Computers follow directions that they are given and that is all that they can do.They cannot interpret what a person means, only what they explicitly and definitely tell it to do. When someone refers to a computer being simple or stupid. They are referring to (usually) an error that has occurred. The Syntax or rules of the language (HTML in this case) have to be followed precisely if we want it to work as intended. Syntax is like Grammar.


Lesson 2: Creating Structure in HTML

Structure

Structure, as it sounds, gives a webpage shape/form. Even when it isn't visible. Structure narrows down the spaces in a web page and help maintain order. In my opinion, creating sound structure or form makes it easier to read code and visualize what is happening. In the previous lesson we discussed Block Elements and it is from them that we create form. In lesson two, the Element used most to create form is the <div> tag.


Developer Tools

In order to see structure easily we need devolper tools. In this lesson we've used the tools available to us in Google Chrome. In Chrome we can right click anywhere on a webpage and select Inspect Element and it will show us the elements of that page or the particular element selected.


The Tree

While using the Inspect Element tool we can see all the elements that make up a page. Each of those elements have a dropdown toggle that reveal more elements that have there own toggles. This can continue until we arrive at the smallest of details. This nesting is sometimes referred to as The Tree.


Text Editors

To write code we need an editor. A text editor of most any kind can do the job. Notepad from PC, one of the simplest editors out there, can do it. These are not the best ones to use though. They would require much more time to use and convert it to a useable HTML page. Some of the editors we've used so far have been CodePen and Sublime Text. These two editors give us more tools and show results faster.


Lesson 3: CSS Style in HTML

What CSS does for us

The one resource that is always in short supply is Time. It takes a while to code a webpage, but without CSS it would take even longer. Repetition is the largest hurdle to tackle while coding a page. Repeating code over and over again to make a page look a certain way takes forever and gives more opportunities for mistakes to happen. With CSS though, we can assign styles to certain Selectors like the <p> element and more (class, id, body, etc) that will affect everything grouped into it.


Inheritance

Inheritance is that which is applied to an element and it's descendants.Descendants only inherit text related assignments, not box.Box meaning things like Background, Border, and Display.

Some Inheritable properties

  1. Color
  2. Line height
  3. Font

This is, essentially, what makes CSS such a valuable tool. It can also be a problem if it isnt done correctly. If the selector is not specific enough the styling will flood over into unwanted areas. To get around this we assign very specific class and id names. for example, if we used (<div class="example"><p>Google is the best browser ever!<p><div>) any change we make in CSS to the example class will apply to everything with that class name.


How to include CSS style into a Webpage
  1. Write CSS in the <head> of your HTML by using <style>div{Code here}</style>.
  2. Write your style inline with your HTML. This is a bad idea because it leads to lots of repeated code and thus extra work. Avoid if possible
  3. Link your HTML to a separate CSS file in the <head>. Link example: <link rel="stylesheet" type="text/css"href="yourfile.css">. The "yourfile.css" must be in the same folder as the HTML document.

While the other options are useful to know, we will mainly be using the third option. It cleans up our coding area by making everything easier to find and read.


Styling

After the CSS file has been linked to the HTML document we can add the style to whichever elements we specify via the element tag (to broad most times) or the class and id names(preferred).

Using an example from earlier (<div class="example"><p>Google is the best browser ever!<p><div>). I want the text for everything with that class name to be blue. The line needed for that in CSS will look like .example{color: blue;}. From this line ".example" is called the Selector while everything between the"{}" is referred to as the Declaration.

Remember the selector refers to an element or an id/class name within an element in your HTML code. This leaves "color:",which is a property, and "blue;" which is a value. There can be multiple declarations inside a selector and there are many different properties and values.
